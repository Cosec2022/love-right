# 统一评分模型

## 12项底层维度

- `security`：依恋安全
- `spark`：心动强度
- `idealism`：浪漫理想
- `reassurance`：确认需求
- `autonomy`：边界独立
- `commitment`：长期投入
- `directness`：沟通直接
- `care`：照顾投入
- `discernment`：识人谨慎
- `novelty`：新鲜偏好
- `validation`：公开认可与被选择需求
- `repair`：关系修复意愿

所有故事尽量沿用同一套底层维度，才能在未来形成跨故事 Love Profile。单篇故事可以只重点覆盖其中一部分，但不得擅自改变相同 ID 的含义。

## 评分与归一化

每个选择对若干维度赋予通常为 `-2` 至 `+2` 的影响。

分支场景使用相同 `slot`。系统对每个 slot 的所有变体寻找该维度最大绝对影响，再累加为理论最大值。最终标准化至 8—92。

这样能保证：

- 选择不同路线不会因为题数不同而天然偏高
- 新增场景变体不会破坏原有路线
- 相同底层维度可以在不同故事中使用相同分制

## 原型匹配

系统计算用户12维向量与各原型目标向量的加权距离，选择距离最小的原型。原型只是结果叙事入口，完整报告仍由维度规则动态拼装。
