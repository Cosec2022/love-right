# 对象魅力个性化系统

## 目标

用户反馈的重点不是简单增加“帅、温柔、霸总”等标签，而是让她感觉：

> 这个人好像慢慢长成了我会喜欢的样子，但故事没有直接告诉我它在迎合我。

## 做法

每篇故事在第3幕左右设置一组自然注意点。用户选择的不是公开类型名称，而是她在当时最容易记住的细节：

```text
稳定：压力下先处理问题
幽默：紧张时让气氛变轻
温柔：照顾但不越过边界
直接：明确表达偏向，不让人一直猜
```

系统保存：

```json
{"appeal": "steady | witty | gentle | direct"}
```

## 三次回响

1. **第6幕**：第一次私人靠近，对象使用相应方式给出邀请或退路。
2. **第12幕**：关系开始明朗，对象以相应方式表达心意。
3. **第17幕**：临近终局，对象在关键行动中再次呈现同一种魅力。

同一种风格不能每次重复相同句式。例如“温柔”不等于每次递外套，而是持续表现为：先询问、保留拒绝权、帮助停在舒服的位置。

## 结果回响

“真正适合你的对象”会加入与 `appeal` 对应的行为描述，但不写：

> 因为你第3题选了C，所以你喜欢温柔男生。

而写成：

> 你容易对细节里的温柔动心，尤其是对方照顾你，却不借照顾要求亲近。

## 禁止事项

- 不让所有风格都变成同一种完美人设
- 不用控制、吃醋、强迫制造所谓强势魅力
- 不把拒绝解释为欲擒故纵
- 不因为用户选择某种风格就改写事实逻辑
- 不让对象魅力选项决定最终人格；它只改变人物呈现和适配对象描述
