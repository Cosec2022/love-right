# Story 08 连续性重写审稿记录

## 原问题

旧版本的婚礼前夜拥有很强的钩子，但选项经常只改变一句心理余波：玩家不下楼、不去桥边、不听解释，下一幕仍会被写到桥下；玩家拒绝接吻，后面也没有明确说明亲密是否停止。更严重的是，Story 08 的后段 `variants` 混入了 Story 07 的许妍、工作台和渡船文案，路线表面记住了 flag，实际记错了另一个故事。

时间线也过于方便：婚礼前夜从酒店到桥下、深夜器材店、录到清晨，再回酒店处理婚礼，缺少伴娘、家人、交通和第二天流程。许承只承担“现实来电”的功能，周叙则因为带来真相自动获得叙事优势。

## 这次怎样补桥

- 明确时间从婚礼前夜 21:30 开始，伴娘在隔壁，次日 06:00 化妆。
- 周叙三天前因项目回城，当天下午才得知婚礼并拿回修好的录音机；这解释“为什么现在”，但正文明确承认他亲自出现仍有自私。
- Slot 1 的见面、前台转交、先看照片、完全不回复都能进入不同的实际接触状态。
- Slot 4 的四条路线分别为：公开桥头同行、留在酒店远程收文件、分开叫车只到监控区、酒店大堂质问。四条 Slot 5 正文完全不同。
- 不去桥边的路线通过前台授权、原始文件、维修单和邮件继续，不会把玩家写到桥下。
- 拒绝解释的路线只收到可核对材料；后文不再假装玩家听过完整独白。
- 深夜器材店改为酒店值班影音工程师和透明商务中心；玩家可要求周叙离开，维修仍能完成。
- Slot 9 分为同空间与远程两类。询问“吻完怎么办”不会被系统偷换成同意；拒绝后正文明确没有第二次试探。
- 第十七轨用六年前的清晨加今晚雨声完成，不再强迫所有路线陪到天亮。
- 许承先确认安全，有共同生活和知情权；隐瞒会在大堂、仪式和婚后继续留下代价。
- Slot 15–17 将延期、取消、继续婚礼、迟到坦白分别写成真实事实。Slot 18 只解释意义，不能再用最后一个按钮改写前面的婚礼决定。

## 五条敌对路线阅读结果

### 1. 完全不见、不去、不听

路线：前台收盒子 → 退回录音机 → 不去桥边 → 只收材料 → 结束当晚接触。

玩家始终没有和周叙进入私人空间。桥声、维修和时间线通过前台、工程师和邮件完成。后续仍能发展，是因为录音和婚礼本身需要处理，而不是因为男主把拒绝当成欲擒故纵。

### 2. 前期靠近，后来拒绝接吻

路线：下楼 → 同行到公开桥头 → 听完整事实 → 在工程师在场时协助 → 拒绝接吻。

Slot 10 明确写周叙退回另一侧，没有第二次试探，后续只谈文件、合同和婚礼。此前靠近没有被系统当作永久许可。

### 3. 前期拒绝，后期逐步开放

路线：不回复 → 只问事实 → 不去桥边、改在大堂谈 → 追问动机 → 允许公开维修 → 先问亲密后果 → 三个月后再见。

重新靠近来自新的事实和新的条件：公开场所、第三人在场、完整合同、明确冷却期。不是旁白突然宣布玩家其实还爱他。

### 4. 接吻后仍选择独处

路线：公开桥头见面 → 主动接吻后退开 → 对许承坦白 → 取消婚礼 → 独自生活。

故事承认吻真实发生，也承认欲望不等于复合。最终独处不会被结算页改写成“其实选择了周叙”。

### 5. 先隐瞒，婚后补全真相

路线：接吻 → 对许承说已睡 → 隐瞒并继续仪式 → 婚后补全真相。

许承通过伴娘发现谎言，婚礼继续也不会清除这条记录。迟到坦白允许故事继续，但结果明确保留婚姻可能暂停和信任受损的代价。

## 为什么没有扩成四倍体量

轻选择仍会快速汇合，但用 aftermath 和 scene variants 记住语气与信息差；真正改变地点、接触方式、身体边界和婚礼事实的选择才增加独立场景。最终为 29 个场景、18 个选择槽位：比旧版多 8 个节点，但没有把每个选项复制成四条完整世界线。

## 仍保留的风险

- Story 08 仍使用 legacy per-story archetype 模型；结果空间问题应在 v0.8 全局模型迁移时单独解决。
- 婚礼前夜题材天然高度戏剧化，现实感提高后仍需用户实读，确认节奏是否过于沉重。
- 部分软选项的向量仍沿用既有 16 维合同，本次只按真实语义校正，没有以 audit 配额反推剧情。
