# Emotional Continuity Engine

## 两套状态分工

- `rawTraits / 16D spatial`：回答“用户在关系里通常怎样选择”。
- `relationship`：回答“这段具体故事此刻是什么温度”。

关系温度包含：

- `warmth`：愿意靠近的热度
- `trust`：愿意相信和交付的程度
- `guard`：保护边界、保留退出权
- `hurt`：失望、被忽略、被替代的感受
- `tension`：未解决冲突与比较压力
- `repair`：解释、回到谈话、共同修复的可能

每个选项会从原有 16D 向量推导关系变化，也允许内容作者用 `relationship` 覆盖个别值。

## `aftermath`

高情绪选择可写：

```json
{
  "id": "d",
  "text": "装作不在意，回复得比他更冷淡",
  "aftermath": "你用冷淡保护自己，也让两个人之间的距离真正拉开。之后每一次靠近，都需要比以前更多的诚意。"
}
```

该文本会作为下一幕的第一段出现。它不是结果解释，而是剧情必须记住的上一幕余波。

## 内容纪律

1. 拒绝后不得自动恢复亲密。
2. 冲突后不得直接跳到浪漫互动。
3. 用户冷淡时，角色只能察觉、询问或保持距离，不能默认她/他在欲擒故纵。
4. 解释不自动等于修复；用户可以接受、保留判断或退出。
5. 回退必须恢复全部情绪状态。
