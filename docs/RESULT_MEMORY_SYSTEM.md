# Love Right Result Memory System v0.4.2

## 用户反馈

故事性和连续剧情让用户愿意完成测试，详细解读也有价值；但原结果页同时展示8项参数和16项底层维度，分数彼此接近，用户完成后难以复述自己的结果。

## 设计原则

```text
后台复杂：16维空间、情境矩阵、轨迹、交互、原型匹配
前台简单：1个名字 + 1个矛盾 + 3个结论 + 4个关系动作
```

底层空间没有删除。它继续负责区分相似选择、不同动机、前后变化和跨故事复用。v0.4.2 新增的是结果压缩层 `result-compressor.js`。

## 用户首先看到什么

1. **一个可复述的结果名称**
   - 只会被行动打动的人
   - 等一句确定的人
   - 会把细节记很久的人
   - 喜欢就不想一直猜的人
   - 靠近了也不想失去自己的人
   - 最想被坚定选择的人
   - 总想把关系照顾好的人
   - 越心动越装作不在意的人

2. **一个核心矛盾**

   不再复述“高边界、高确认需求”等术语，而是把两个相互牵制的需要写成一句生活化判断。

3. **三个关键结论**
   - 会被什么打动
   - 在关系里最怕什么
   - 真正适合什么样的人

4. **四个关系动作**
   - 怎么心动
   - 怎么靠近
   - 怎么自保
   - 怎么留下

这些动作由原8项派生指标组合生成，但不显示分数。

## 剧情证据

旧版按数值强度列出4个选择，并显示轴向加减。新版只保留3个剧情瞬间：

```text
第一次靠近 / 亲密
→ 模糊、竞争或冲突
→ 是否愿意进入承诺
```

每个瞬间用故事语言解释其意义，不再显示 `approach +0.8` 一类内部权重。

## 详细内容

以下内容继续保留：

- 故事结局
- 恋爱心理长文
- 过往关系投射
- 未来三幕
- 适配对象详细说明
- 容易误判的信号

16维底层数据只放在默认折叠的“研究模式”中，并明确说明它不是用户需要记住的结果。

## 内容覆盖

默认8类记忆结果由引擎提供。未来某个故事需要特殊表达时，可以在 `results.json` 中增加可选的 `memoryProfiles`，按 archetype id 覆盖：

```json
{
  "memoryProfiles": {
    "steady-confirm": {
      "title": "等一句确定的人",
      "hook": "……",
      "contradiction": "……",
      "attraction": "……",
      "fear": "……",
      "fit": "……"
    }
  }
}
```


## v0.4.3：六项相对情感指数

用户反馈表明，完全移除指数会让结果缺少可快速比较的抓手。因此结果页恢复6项维度，但不重新暴露全部16维底层参数。

六项指数按用户在本篇故事中的相对强弱做自适应对比。该方法用于突出“你自己内部哪些倾向更强、哪些更弱”，不是跨用户常模或专业心理分数。

结果页先展示故事结局，再进入心理解释，使答题时积累的剧情情绪不会被参数卡片打断。
