# 故事内容制作规范

## 正式故事规模

```text
18 个 assessment slots
每幕 4 个选择为主
2—4 条显著路线
4 个终局 outcome
完成时间 8—12 分钟
```

分支场景可以增加实际场景数量，但同一心理位置使用同一个 slot。

## 每一幕完成三件事

1. 关系发生新变化。
2. 用户面对真实策略取舍。
3. 答案改变空间向量，必要时改变剧情或 flags。

## 真实分支

以下行为必须留下后果：

- 拒绝、离开、拉黑
- 接受或拒绝接触
- 直接确认关系
- 第三人或旧关系
- 失约、失联和冲突
- 事业、家庭与异地
- 明确不愿继续

禁止：

```text
用户拒绝 → 对方仍默认同行
用户离开 → 下一幕仍默认约会
用户问清关系 → 后文假装没有问过
四个选项 → 完全相同的心理方向
```

分支可以重新汇合，但人物必须记得发生过什么。

## 对象魅力

对象需要有吸引力，但不能完美到失真。

推荐把魅力拆成：

```text
能力：遇事能处理，不抢走用户的位置
幽默：缓解压力，不拿用户脆弱开玩笑
温柔：先询问，照顾不附带亲近要求
直接：说明偏爱，也保留用户拒绝权
```

用户隐含选中的风格在后文逐渐加强。不要直接写“你选了温柔型，因此他变温柔”。

## 选择向量

示例：

```json
{
  "text": "告诉他，我确实有一点在意",
  "context": "jealousy",
  "intensity": 1,
  "confidence": 0.95,
  "vector": {
    "expression": 0.72,
    "jealousy": 0.38,
    "certainty": 0.32,
    "trust": 0.15
  }
}
```

规则：

- 值域 `[-1,1]`
- 单个选项通常影响3—6项
- 正负都表示证据，不表示好坏
- 不为制造结果平衡而写不符合语义的权重
- 同一选项不要同时把互相矛盾的轴都推到极端

## 结果文案

好的结果顺序：

```text
先说核心需要
再写表面行为与内在动机的差异
引用关键选择
推演过去可能形成的模式
给出未来关系画面
描述适合对象的行为
提醒容易误判的信号
```

使用：

```text
可能 / 往往 / 更容易 / 在这个故事里
```

不得声称：

- 准确读取前任数量或创伤
- 诊断依恋障碍或人格障碍
- 预测具体日期和人物
- 判断忠诚度、道德或智力
