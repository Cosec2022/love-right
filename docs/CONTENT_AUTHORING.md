# 故事内容制作规范

## 一篇正式故事的建议规模

```text
14—22 个 assessment slots
每幕 2—4 个选择
2—4 条显著路线
3—6 个终局 outcome
完成时间 6—12 分钟
```

## 每一幕必须完成三件事

1. 剧情发生了新的变化。
2. 用户面临真实取舍，而不是四种同义说法。
3. 选择至少影响一个底层维度，必要时改变后续剧情。

## 真实分支规则

以下行为必须在剧情中产生后果：

- 拒绝、离开、拉黑
- 直接询问关系
- 接受或拒绝身体接触
- 发现欺骗或第三者
- 选择事业、家庭或异地
- 明确不愿继续

禁止出现：

```text
用户拒绝 → 对方仍默认同行
用户选择离开 → 下一幕仍默认约会
用户要求解释 → 后文假装没问过
四个选项 → 全部进入完全相同的台词且无评分差异
```

共享场景可以重新汇合，但人物必须记得此前发生的事。需要时使用 `flags`、场景变体和条件跳转。

## 选择的写法

四个选项最好分别表达不同心理策略：

```text
主动确认
含蓄试探
观察事实
退出并保护边界
```

不是：

```text
开心地答应
害羞地答应
犹豫后答应
想了想还是答应
```

## 评分权重

推荐单项影响：

```text
-2  明显反向证据
-1  轻微反向证据
 0  无关
+1  轻微正向证据
+2  明显正向证据
```

单个选择不要给所有维度加分。通常影响 2—4 项即可。

## 结果写法

结果需要：

- 先说用户的核心需要
- 再说明行为表现和内在动机可能不同
- 用“可能、往往、更容易”表达推演
- 给出来自关键选择的证据
- 把适配对象写成行为特征，不写星座、职业或财富保证

结果不能声称：

- 准确读取真实前任数量
- 诊断依恋障碍或人格障碍
- 确定预测何时遇见某个人
- 根据少量选择判断忠诚度、道德或智力
